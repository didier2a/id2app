# Spécification lourde ultra détaillée — INFOSERV2A Cyber & Réseau

**Objet :** cahier des charges structuré pour soumission au moteur ID2APP V6 Premium Quality Engine.  
**Sources visuelles à interpréter :**

1. Infographie stratégique : **« INFOSERV2A : Le Pivot Stratégique vers l’Expertise B2B Premium »**.
2. Infographie d’architecture de présence web : **« INFOSERV2A : L’Architecture d’une Présence Web Premium »**.
3. Charte graphique / logo : **« INFOSERV2A Cyber & Réseau »**.

**But :** produire un site premium B2B, cyber/réseau, local, sobre, crédible, ultra ciblé, sans contamination par d’anciens modèles grand public, sans esthétique hacker caricaturale, et en utilisant exclusivement le logo INFOSERV2A fourni comme identité visuelle.

---

## 1. Résumé exécutif

Le site à produire doit matérialiser le pivot stratégique d’INFOSERV2A vers une offre **B2B premium** centrée sur :

- la cybersécurité locale ;
- l’audit terrain ;
- la vérification physique sur site ;
- les réseaux critiques ;
- le WiFi professionnel ;
- Starlink ;
- les liaisons difficiles ;
- la continuité numérique pour entreprises du Sud Corse.

Le site ne doit plus évoquer l’ancien modèle de dépannage informatique quotidien, d’assistance grand public, de réparation PC, d’imprimantes ou de maintenance généraliste.

La promesse centrale est :

> **La sécurité doit être vérifiée sur site.**

La formulation complémentaire peut être :

> **Votre informatique peut être gérée à distance. Votre sécurité doit aussi être vérifiée sur site.**

Le site doit s’adresser prioritairement aux **TPE/PME, décideurs locaux, dirigeants, responsables de site, hôtels, campings, cabinets médicaux, professions juridiques, immobilier, villas/résidences haut de gamme, commerces et sites isolés**.

Le rendu doit être :

- sombre ;
- cyber premium ;
- précis ;
- millimétré ;
- local ;
- rassurant ;
- non anxiogène ;
- techniquement crédible ;
- B2B ;
- orienté qualification.

## 2. Interprétation stratégique des infographies

### 2.1 Pivot principal

L’infographie « Pivot Stratégique » oppose deux modèles.

#### Ancien modèle à rejeter totalement

L’ancien modèle est caractérisé par :

- assistance quotidienne ;
- dépannage général ;
- grand public ;
- imprimantes ;
- icônes tournevis / réparateur ;
- matériel cassé ;
- ton informel ;
- image de prestataire généraliste ;
- posture de maintenance informatique classique.

Ces éléments doivent être interdits dans le site final.

#### Nouveau modèle à imposer

Le nouveau modèle est :

- autorité B2B ;
- expertise cyber et réseau ;
- focus TPE/PME ;
- décideurs locaux ;
- cybersécurité ;
- réseaux critiques ;
- interface chic et ultra-tech ;
- ancrage Porto-Vecchio / Sud Corse ;
- qualité premium ;
- approche terrain.

Le site doit donc incarner une rupture claire :

> INFOSERV2A n’est plus perçu comme un dépanneur informatique. INFOSERV2A devient une autorité locale de vérification cyber et réseau pour entreprises exposées ou dépendantes de leur infrastructure numérique.

### 2.2 Angle mort stratégique

L’infographie identifie un point central :

> L’interface doit rejeter l’image du prestataire à distance classique pour “crier” la réalité physique de l’audit local.

Traduction pour le site :

- le site doit montrer la présence locale ;
- le terrain doit être visible dans le vocabulaire ;
- l’audit ne doit pas être abstrait ;
- il faut parler de site, local, vérification, cartographie, contrôle, infrastructure, dépendances, risques concrets ;
- l’entreprise doit être perçue comme capable de se déplacer, observer, contrôler, documenter et expliquer.

### 2.3 Architecture d’information

L’infographie propose une structure volontairement resserrée autour de 7 pages publiques principales :

1. Accueil
2. Cyber
3. Réseau
4. Secteurs sensibles
5. Méthode
6. À propos
7. Contact

À cela doivent s’ajouter les pages légales :

8. Mentions légales
9. Confidentialité

Le site ne doit pas créer de pages d’assistance grand public, réparation, maintenance, dépannage, aide PC ou support domestique.

### 2.4 Conversion et qualification

Le site doit fonctionner comme un **entonnoir de qualification strict**.

L’objectif n’est pas d’obtenir un maximum de demandes, mais d’obtenir des demandes pertinentes :

- dirigeants ;
- entreprises ;
- sites critiques ;
- besoins réseau/cyber ;
- projets en Corse-du-Sud ;
- demandes suffisamment qualifiées.

Le site doit donc dissuader implicitement les demandes hors cible : réparation PC, imprimantes, particuliers, assistance immédiate non qualifiée, dépannage domestique.

Le site ne doit pas être publié comme `completed` si l’un des points suivants est vrai :

- page principale de moins de 500 mots ;
- pages trop similaires ;
- paragraphes génériques répétés ;
- phrase “Cette section présente” ;
- H1 vide ;
- même meta description sur plusieurs pages ;
- absence de termes métier propres à chaque page.

### 13.3 Contamination

Échec si présence dans pages publiques de :

- PortoFlow ;
- GEM ;
- Casa di l’Isula ;
- ID2APP visible ;
- template_ ;
- REPLACE_ ;
- dépannage grand public ;
- réparation PC ;
- imprimante ;
- hacker à capuche.

### 13.4 CTA

Échec ou warning si CTA principal contient :

- Contactez-nous ;
- Nos services ;
- Besoin d’aide ?

- cyber.html ;
