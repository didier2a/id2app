Présentation du test rapide V4

Comprendre les objectifs, le périmètre et la logique du workflow ID2App V4 en quelques points essentiels.

Pourquoi ce test ?

Le micro-site V4 Smoke Test a été conçu pour valider rapidement l'ensemble du workflow ID2App V4 dans des conditions réelles. Il couvre la génération de pages HTML statiques, l'intégration des styles, la navigation inter-pages et la compatibilité GitHub Pages.

Chaque page de ce site représente un segment du pipeline de génération : de la spécification initiale jusqu'au déploiement final. L'objectif est de détecter les points de friction et de confirmer la cohérence du rendu sur l'ensemble des 8 pages.

✦

Périmètre défini

8 pages statiques, un seul fichier CSS partagé, un seul fichier JavaScript vanilla, aucune dépendance externe.

✦

Validation complète

Navigation active, accessibilité de base, responsive mobile, liens relatifs corrects et déploiement GitHub Pages.

✦

Résultat attendu

Un site fonctionnel, cohérent visuellement et maintenable, généré intégralement par le pipeline ID2App V4.
