Questions fréquentes

Retrouvez ici les réponses aux questions les plus courantes sur le workflow ID2App V4 et ce micro-site de test.

Toutes les questions

ID2App V4 est un workflow de génération de sites et d'applications à partir d'une spécification structurée. Il produit des fichiers HTML, CSS et JavaScript prêts à déployer, compatibles GitHub Pages.

Ce micro-site est un smoke test : il permet de valider rapidement que l'ensemble du pipeline de génération fonctionne correctement, de la spécification jusqu'au déploiement final.

Oui. Le site utilise du HTML5 sémantique, du CSS moderne avec variables et du JavaScript vanilla sans dépendance externe. Il est compatible avec tous les navigateurs modernes et fonctionne sans build step.

Il suffit de pousser les fichiers dans un dépôt GitHub et d'activer GitHub Pages depuis les paramètres du dépôt. Aucune configuration supplémentaire n'est nécessaire car tous les liens sont relatifs.

Absolument. Le fichier styles.css utilise des variables CSS pour les couleurs, typographies et espacements. Il suffit de modifier ces variables pour adapter l'ensemble du site en quelques lignes.

Oui. Le site suit les bonnes pratiques WCAG : balises sémantiques, attributs ARIA, contrastes suffisants, navigation au clavier et indicateurs de focus visibles sur tous les éléments interactifs.
