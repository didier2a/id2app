Vos questions, nos réponses

ID2App V4 est un workflow de génération automatisée de fichiers pour des projets web statiques. Il permet de produire rapidement des sites HTML/CSS/JS structurés, accessibles et compatibles GitHub Pages à partir d'une spécification de projet.

Ce micro-site est un smoke test conçu pour valider rapidement l'ensemble du pipeline V4 : génération des fichiers, cohérence de la navigation, qualité du code produit et compatibilité avec un hébergement statique comme GitHub Pages.

Oui. Chaque page est construite avec du HTML5 sémantique, des attributs ARIA appropriés et un CSS responsive basé sur des variables et des media queries. La navigation mobile est gérée via JavaScript vanilla sans dépendance externe.

Absolument. Le site utilise uniquement des fichiers statiques avec des liens relatifs. Il suffit de pousser le dossier du projet dans un dépôt GitHub et d'activer GitHub Pages depuis les paramètres du dépôt.

Le projet repose exclusivement sur HTML5, CSS3 avec variables personnalisées, et JavaScript vanilla. Aucune bibliothèque ou framework externe n'est requis, ce qui garantit légèreté et rapidité de chargement.

Il suffit de modifier les variables CSS dans styles.css pour adapter la palette de couleurs, de remplacer les contenus textuels dans chaque fichier HTML, et d'ajuster les comportements dans script.js selon les besoins du projet cible.
